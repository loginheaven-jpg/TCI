# TCI 그룹 분석 서비스 - Phase 2 종합 기획서

> **Version 2.0** | 2026-01-06
> 
> Phase 1 프로토타입 기반 + Supabase 연동 + 개인진단지 대폭 보완

---

## 1. 프로젝트 개요

### 1.1 목적
코칭 전문가가 다수의 TCI 검사 결과를 그룹 단위로 분석하고, 개인별 심층 진단 리포트를 생성하는 웹 애플리케이션.

### 1.2 Phase 2 핵심 변경사항

| 영역 | Phase 1 | Phase 2 |
|-----|---------|---------|
| **데이터 저장** | LocalStorage (브라우저 종속) | Supabase PostgreSQL (클라우드) |
| **사용자 관리** | 없음 | Supabase Auth (이메일/소셜 로그인) |
| **개인 리포트** | 5열 테이블 + 간단 해석 | **유형 기반 심층 분석** + 코칭 가이드 |
| **유형 분류** | 없음 | 기질 27유형 + 성격 27유형 + 성격장애 경향성 |
| **해석 데이터** | 하드코딩 | Supabase에서 동적 로드 |

## 3. Supabase 데이터베이스 스키마

### 3.1 ERD 개요

```
┌─────────────┐     ┌─────────────┐     ┌─────────────────┐
│   users     │────<│   groups    │────<│    members      │
└─────────────┘     └─────────────┘     └─────────────────┘
                                               │
                    ┌─────────────┐            │
                    │ interpretations │<───────┘
                    │ (해석 데이터)    │
                    └─────────────┘
```

### 3.2 테이블 정의

#### users (사용자)
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  organization TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

#### groups (그룹)
```sql
CREATE TABLE groups (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_groups_user_id ON groups(user_id);
```

#### members (참가자 - TCI 데이터)
```sql
CREATE TABLE members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  group_id UUID REFERENCES groups(id) ON DELETE CASCADE,
  
  -- 기본 정보
  name TEXT NOT NULL,
  gender TEXT CHECK (gender IN ('남', '여')),
  age INTEGER,
  
  -- 상위지표 원점수
  ns INTEGER, ha INTEGER, rd INTEGER, ps INTEGER,
  sd INTEGER, co INTEGER, st INTEGER,
  
  -- 상위지표 T점수
  ns_t INTEGER, ha_t INTEGER, rd_t INTEGER, ps_t INTEGER,
  sd_t INTEGER, co_t INTEGER, st_t INTEGER,
  
  -- 상위지표 백분위
  ns_p INTEGER, ha_p INTEGER, rd_p INTEGER, ps_p INTEGER,
  sd_p INTEGER, co_p INTEGER, st_p INTEGER,
  
  -- 하위지표 원점수 (NS)
  ns1 INTEGER, ns2 INTEGER, ns3 INTEGER, ns4 INTEGER,
  -- 하위지표 원점수 (HA)
  ha1 INTEGER, ha2 INTEGER, ha3 INTEGER, ha4 INTEGER,
  -- 하위지표 원점수 (RD)
  rd1 INTEGER, rd2 INTEGER, rd3 INTEGER, rd4 INTEGER,
  -- 하위지표 원점수 (PS)
  ps1 INTEGER, ps2 INTEGER, ps3 INTEGER, ps4 INTEGER,
  -- 하위지표 원점수 (SD)
  sd1 INTEGER, sd2 INTEGER, sd3 INTEGER, sd4 INTEGER, sd5 INTEGER,
  -- 하위지표 원점수 (CO)
  co1 INTEGER, co2 INTEGER, co3 INTEGER, co4 INTEGER, co5 INTEGER,
  -- 하위지표 원점수 (ST)
  st1 INTEGER, st2 INTEGER, st3 INTEGER,
  
  -- 유형 코드 (자동 계산)
  temperament_type TEXT,  -- 기질유형 (예: HML, LHH)
  character_type TEXT,    -- 성격유형 (예: HHL, LLH)
  
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_members_group_id ON members(group_id);
```

#### interpretations (해석 데이터)
```sql
CREATE TABLE interpretations (
  id SERIAL PRIMARY KEY,
  category TEXT NOT NULL,  -- 'temperament', 'character', 'subscale', 'interaction'
  type_code TEXT NOT NULL, -- 'HML', 'NS-높음', 'NH' 등
  type_name TEXT,          -- '독립적인-자유주의적인', '높은 자극추구' 등
  description TEXT NOT NULL,
  strengths TEXT,
  weaknesses TEXT,
  coaching_tips TEXT,
  personality_disorder_tendency TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_interpretations_category ON interpretations(category);
CREATE INDEX idx_interpretations_type_code ON interpretations(type_code);
```

## 4. 개인 진단 리포트 대폭 보완

### 4.1 기존 vs 신규 비교

| 구분 | Phase 1 | Phase 2 |
|-----|---------|---------|
| **기질 분석** | 7개 척도 수치만 표시 | **27개 유형별 심층 해석** |
| **성격 분석** | 수치만 표시 | **27개 유형별 심층 해석** |
| **유형 판정** | 없음 | L/M/H 3단계 자동 분류 |
| **상호작용 해석** | 없음 | NS-HA, NS-RD, HA-RD 조합 해석 |
| **성격장애 경향성** | 없음 | 자동 판별 및 경고 |
| **코칭 가이드** | 없음 | 유형별 맞춤 코칭 포인트 |

### 4.2 유형 분류 기준 (T점수/백분위 기반)

```javascript
// T점수 기준
const getLevel = (tScore) => {
  if (tScore < 45) return 'L';  // Low
  if (tScore <= 55) return 'M'; // Medium  
  return 'H';                    // High
};

// 백분위 기준 (대안)
const getLevelByPercentile = (percentile) => {
  if (percentile < 30) return 'L';
  if (percentile <= 70) return 'M';
  return 'H';
};
```

### 4.5 성격장애 경향성 판별

```javascript
// 성격장애 경향성 자동 판별 로직
const checkPersonalityDisorderTendency = (sd_p, co_p) => {
  // 조건: SD 백분위 < 30 AND CO 백분위 < 30
  // 또는 SD + CO 합산 백분위 < 30
  
  if ((sd_p < 30 && co_p < 30) || (sd_p + co_p) < 30) {
    return {
      warning: true,
      message: "자율성 및 연대감의 백분위 점수가 모두 30점 미만이거나 " +
               "합산 백분위 점수가 30점 미만인 경우, 적응상의 어려움을 보이고 " +
               "미성숙하며 성격장애를 보일 가능성이 높다고 평가됨."
    };
  }
  
  return { warning: false, message: null };
};
```

### 4.6 기질 상호작용 해석

#### NS × HA (자극추구 × 위험회피)
| NS | HA | 유형 | 해석 |
|----|----|----|------|
| H | H | NH | 접근-회피 갈등, 신경증적 경향, 만성적 좌절감 |
| L | L | nh | 명랑, 태평, 자신감, 부정적 감정으로부터 자유 |
| H | L | Nh | 충동적, 외향적, 공격적, 스릴 추구 |
| L | H | nH | 내향적, 안전 지향, 내면의 정신생활 추구 |

#### NS × RD (자극추구 × 사회적민감성)
| NS | RD | 유형 | 해석 |
|----|----|----|------|
| H | H | NR | 사교적, 주목 끄는 행동, 감정 잘 드러냄 |
| L | L | nr | 관계보다 개인생활 중시, 잘 드러나지 않음 |
| H | L | Nr | 기회주의적, 비관습적, 진실성 부족 |
| L | H | nR | 보수적, 전통적, 양심적, 진솔 |

#### HA × RD (위험회피 × 사회적민감성)
| HA | RD | 유형 | 해석 |
|----|----|----|------|
| H | H | HR | 수동적, 회피적, 복종적, 의존적 |
| L | L | hr | 관계 두려워하지 않으나 무관심, 반항적 |
| H | L | Hr | 회피와 고립 선택, 대인 영향력 작음 |
| L | H | hR | 친화적, 사교적, 대인 영향력 큼 |

### 4.7 기질 유형별 성격장애 경향성 연결

| 기질유형 | 성격장애 경향성 |
|---------|----------------|
| HHL (폭발적인) | 경계선(Borderline) |
| HLL (모험적인) | 반사회성(Antisocial) |
| LHH (조심스러운) | 회피성(Avoidant) |
| HLH (열정적인) | 연극성(Histrionic) |
| LHL (조직적인) | 강박성(Obsessional) |
| HHH (예민한) | 자기애성(Narcissistic) |
| LLL (독립적인) | 조현성(Schizoid) |
| LLH (신뢰로운) | 안정된(Staid) |

### 4.8 인내력(PS)의 역할

| 수준 | 장점 | 단점 |
|-----|------|------|
| **높음** | 많은 성취, 강한 회복탄력성, 긍정적 정서 | 보상 없는 상황에서 정서적 부담, 융통성 부족 |
| **낮음** | 유연하고 실용적, 부정적 정서 적음 | 성취 부족, 회복탄력성 약함, 쉽게 포기 |

```javascript
import { useState, useEffect } from 'react';
import { supabase } from '../lib/supabase';

export const useAuth = () => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    supabase.auth.getSession().then(({ data: { session } }) => {
      setUser(session?.user ?? null);
      setLoading(false);
    });

    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (_event, session) => {
        setUser(session?.user ?? null);
      }
    );

    return () => subscription.unsubscribe();
  }, []);

  const signIn = async (email, password) => {
    const { error } = await supabase.auth.signInWithPassword({ email, password });
    return { error };
  };

  const signUp = async (email, password, name) => {
    const { error } = await supabase.auth.signUp({
      email, password,
      options: { data: { name } }
    });
    return { error };
  };

  const signOut = async () => {
    await supabase.auth.signOut();
  };

  return { user, loading, signIn, signUp, signOut };
};
```

### 6.3 그룹 CRUD 훅

```javascript
// src/hooks/useGroups.js
import { useState, useEffect } from 'react';
import { supabase } from '../lib/supabase';

export const useGroups = (userId) => {
  const [groups, setGroups] = useState([]);
  const [loading, setLoading] = useState(true);

  const fetchGroups = async () => {
    setLoading(true);
    const { data, error } = await supabase
      .from('groups')
      .select('*, members (*)')
      .eq('user_id', userId)
      .order('created_at', { ascending: false });

    if (!error) setGroups(data || []);
    setLoading(false);
  };

  const createGroup = async (name, description) => {
    const { data, error } = await supabase
      .from('groups')
      .insert({ name, description, user_id: userId })
      .select()
      .single();

    if (!error) {
      setGroups(prev => [{ ...data, members: [] }, ...prev]);
    }
    return { data, error };
  };

  const deleteGroup = async (groupId) => {
    const { error } = await supabase
      .from('groups')
      .delete()
      .eq('id', groupId);

    if (!error) {
      setGroups(prev => prev.filter(g => g.id !== groupId));
    }
    return { error };
  };

  const addMembers = async (groupId, membersData) => {
    const membersWithTypes = membersData.map(m => ({
      ...m,
      group_id: groupId,
      temperament_type: calculateTemperamentType(m),
      character_type: calculateCharacterType(m)
    }));

    const { data, error } = await supabase
      .from('members')
      .insert(membersWithTypes)
      .select();

    if (!error) {
      setGroups(prev => prev.map(g => 
        g.id === groupId 
          ? { ...g, members: [...(g.members || []), ...data] }
          : g
      ));
    }
    return { data, error };
  };

  useEffect(() => {
    if (userId) fetchGroups();
  }, [userId]);

  return { groups, loading, createGroup, deleteGroup, addMembers, fetchGroups };
};

const calculateTemperamentType = (m) => {
  const getLevel = (t) => t < 45 ? 'L' : t <= 55 ? 'M' : 'H';
  return `${getLevel(m.ns_t)}${getLevel(m.ha_t)}${getLevel(m.rd_t)}`;
};

const calculateCharacterType = (m) => {
  const getLevel = (t) => t < 45 ? 'L' : t <= 55 ? 'M' : 'H';
  return `${getLevel(m.sd_t)}${getLevel(m.co_t)}${getLevel(m.st_t)}`;
};
```

```javascript
// src/utils/migration.js
export const migrateFromLocalStorage = async (userId) => {
  const localData = JSON.parse(localStorage.getItem('tci-groups') || '[]');
  if (localData.length === 0) return { success: true, migrated: 0 };

  let migratedCount = 0;

  for (const group of localData) {
    const { data: newGroup, error } = await supabase
      .from('groups')
      .insert({ name: group.name, description: group.desc || '', user_id: userId })
      .select()
      .single();

    if (error) continue;

    if (group.members?.length > 0) {
      const membersWithGroupId = group.members.map(m => ({
        ...m,
        group_id: newGroup.id,
        temperament_type: calculateTemperamentType(m),
        character_type: calculateCharacterType(m)
      }));
      await supabase.from('members').insert(membersWithGroupId);
    }
    migratedCount++;
  }

  return { success: true, migrated: migratedCount };
};
```
